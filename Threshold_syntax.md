# ПОРОГИ

## Синтаксис порогов

**Пороги** - это механизм определения наличия и уровня состояния тревоги на устройстве в целом и его составных частях,
наличия и состояния превышения допустимых значений отдельных параметров всех составных частей устройства на основании
анализа полученных в результате опроса значений как параметров самого устройства, так и значений параметров смежных
устройств.

Анализ значений параметров сводится к выполнению логического выражения, которое в общем виде имеет следующее
представление:

```text
IF [Логическое выражение] THEN [Атрибут тревоги [устройства]/[составной части]/[параметра]] = [Установленное значение]
```

_Логическое выражение_ - это программная конструкция, которая принимает только одно из двух возможных значений:
«истина» (`true`) или «ложь» (`false`).

_Атрибут тревоги [устройства]/[составной части]/[параметра]_ - это значение состояния тревоги, определяющее наличие и
критичность тревоги для устройства, его составной части или по текущему значению параметра самого устройства или его
составной части.

Атрибут тревоги (поле "alarm_level" в компоненте или параметре json-объекта результата опроса) может принимать следующие
_Установленные значения_:

- `NONE`/ отсутствие поля - состояние отсутствия тревоги по данному компоненту/параметру;
- `HIGH ALARM` - состояние аварии по верхнему пороговому значению по данному значению параметра;
- `HIGH WARNING` - состояние предупреждения по верхнему пороговому значению по данному значению параметра;
- `ALARM` - состояние аварии на данном компоненте или по данному значению параметра;
- `WARNING` - состояние тревоги на данном компоненте или по данному значению параметра;
- `LOW WARNING`- состояние предупреждения по нижнему пороговому значению по данному значению параметра;
- `LOW ALARM` - состояние аварии по нижнему пороговому значению по данному значению параметра.

## Логические выражения

Логические выражения должны строиться на основании общих правил информатики и содержать от одного до нескольких
унитарных результатов сравнения значения параметра с каким-либо значением соответствующего параметру типа, объединённых
в логическую цепочку с общим результатом.

Каждый унитарный результат сравнения вычисляется на основании выражения, общий вид которого выглядит следующим образом:

```text
[Источник данных] [Логический оператор сравнения] [Величина]
```

### Источник данных

Для однозначного указания на значение конкретного параметра или поля в иерархической структуре оборудования используется
точечная нотация (`.`). Синтаксис поддерживает бесконечную вложенность компонентов, но всегда строго завершается одним
финальным элементом (параметром или полем).

В качестве источника данных для сравнения могут выступать значения параметров и значения служебных полей параметров и
компонентов.

Шаблоны для указания источника данных:

- параметра:

```text
[Протокол]:[host]:[port].[с].[Тип базового компонента]. ...
... [c].[Тип компонента].[Внутренний порядок]. ...
... [c].[Тип компонента].[Внутренний порядок]. ...
... [p].[Название параметра]
```

- поля:

```text
[Протокол]:[host]:[port].[с].[Тип базового компонента]. ...
... [c].[Тип компонента].[Внутренний порядок]. ...
... [c].[Тип компонента].[Внутренний порядок]. ...
... [f].[Название поля]
```

#### Правила построения цепочки

1. _Точка старта_: Путь всегда начинается с идентификатора устройства - сочетания `[Протокол]:[host]:[port]`;
2. _Узел вложения (Компонент)_: Описывается строго тремя элементами через точку -
   `[c].[Тип компонента].[Внутренний порядок]` - где `[c]` - маркер компонента (сокращение от слова _component_),
   который говорит системе, что цепочку следует продолжить вглубь, `[Тип компонента]` - наименование (значение поля
   _title_) компонента в модельном каталоге, внутри которого необходимо осуществить поиск последующих элементов,
   `[Внутренний порядок]` - внутренний порядок компонента внутри родительского компонента;
3. _Бесконечная вложенность компонентов_: Блоки компонентов (`[c].[Тип компонента].[Внутренний порядок]`) могут
   повторяться друг за другом сколько угодно раз, отражая реальную физическую или логическую вложенность
   составных частей оборудования;
4. _Финал пути_: Путь обязан заканчиваться маркером конечной цели (`[p]` — параметр или `[f]` — поле) и
   её `[Названием]`.

Главное ограничение: Параметр или поле в пути может быть только одно и только в самом конце. После маркера `[p]` или
`[f]` продолжать цепочку компонентов запрещено.

### Логический оператор сравнения

В качестве операторов сравнения используются общепринятые в программировании операторы:



### Примеры использования

```text
IF snmp:192.168.31.12:161.c.component.1.f.alarm_level == ALARM THEN snmp:127.0.0.1:161.c.link.23.f = WARNING
```

```text
IF ANY:ANY:ANY.c.component.ANY.c.interface.ANY.f.alarm_level == ALARM THEN THIS:THIS:THIS.c.component.ANY.f = WARNING
```

```text
IF ANY:ANY:ANY.c.component.ANY.c.interface.ALL.f.alarm_level == ALARM THEN THIS:THIS:THIS.c.component.ANY.f = ALARM
```
