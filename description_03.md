## [POST] /api/v1/catalog/link/component-param/{componentId}/{paramId} - Связать компонент с параметром

> [!TIP]
> API предназначен для связывания компонентов и параметров по их id: первым указывается id компонента, вторым - id
> параметра.  
> В результате выполнения запроса будет получен обновлённый компонент.

### Возможные коды ошибок

400: Bad Request - Некорректные форматы переданных идентификаторов в пути
404: Not Found - Переданный компонент или параметр не обнаружены в моделях
500: Internal Server Error - Ошибка СУБД при вставке связи в component_param

## [DELETE] /api/v1/catalog/link/component-param/{componentId}/{paramId} - Удалить связь компонента с параметром

> [!TIP]
> API предназначен для отвязывания параметров от компонентов по их id: первым указывается id компонента, вторым - id
> параметра.  
> В результате выполнения запроса будет получен обновлённый компонент.

> [!IMPORTANT]
> Отвязать параметр, который был унаследован от родителя, невозможно.  
> Чтобы отвязать параметр, который был унаследован от компонента-родителя, нужно отвязывать этот параметр именно от
> компонента-родителя. В этом случае параметр будет отвязан от всех компонентов-потомков, включая целевой
> компонент-потомок.  
> Для того, чтобы узнать, к какому компоненту прикреплён конкретный параметр, необходимо использовать API
> [GET] /api/v1/catalog/param/search/{id}.
